# GreenKode Research Plan: Measuring the Impact of Code Efficiency on Carbon Emissions

## 1. Objective
To empirically validate the hypothesis that algorithmic optimization leads to measurable reductions in energy consumption and carbon emissions, using **GreenKode** as the measurement tool.

## 2. Hypothesis
Optimizing a computational task from $O(n^2)$ (Quadratic) to $O(n)$ (Linear) or $O(n \log n)$ (Log-Linear) complexity will result in a statistically significant reduction in CPU energy usage and CO2 equivalents.

## 3. Methodology

### 3.1. Test Environment
*   **Hardware**: Local Workstation (Intel Core i7-7500U).
*   **OS**: Windows 10/11.
*   **Tool**: GreenKode CLI (`greenkode run`).

### 3.2. Scenarios
We will create two Python scripts performing the same task: **finding duplicate integers in a large list (N=10,000)**.

#### Scenario A: The "Dirty" Code (Inefficient)
*   **Algorithm**: Nested loops to compare every element with every other element.
*   **Complexity**: $O(n^2)$.
*   **File**: `tests/scenarios/dirty_code.py`.

#### Scenario B: The "Green" Code (Optimized)
*   **Algorithm**: Using a hash set to track seen elements.
*   **Complexity**: $O(n)$.
*   **File**: `tests/scenarios/green_code.py`.

### 3.3. Data Collection
*   Execute each scenario **5 times** to account for OS background noise.
*   Metrics to record:
    *   **Duration (s)**: Execution time.
    *   **Energy (kWh)**: CPU energy consumed.
    *   **Emissions (kgCO2eq)**: Carbon footprint.
*   Data source: `emissions.csv` generated by GreenKode.

## 4. Analysis Plan
1.  Aggregate data from `emissions.csv`.
2.  Calculate **Mean** and **Standard Deviation** for both scenarios.
3.  Compute **Percentage Reduction**:
    $$ \text{Reduction} = \frac{\text{Dirty} - \text{Green}}{\text{Dirty}} \times 100\% $$
4.  Generate a comparative table and bar chart (simulated in LaTeX).
